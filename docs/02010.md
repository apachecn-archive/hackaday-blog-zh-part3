# 本周失败:总是检查保险丝

> 原文:[https://hacka day . com/2016/04/06/每周故障检查保险丝/](https://hackaday.com/2016/04/06/fail-of-the-week-always-check-the-fuse/)

[Tomas]在瑞典的 Umeå Hackerspace，有一些损坏的音频设备，包括一个 Sharp CD 播放器/放大器。当他试图解决问题的时候出了什么问题，这是一个失败的故事，我们都可以从中吸取教训。

该设备在打开后工作了大约一秒钟，然后自动关闭。这是一个充满希望的信号，是时候开始调试了。他将小信号和逻辑板从电路中取出，只留下电源和放大器，并施加了电流。

神奇的蓝色烟雾随之而来，来自放大器。缺少[合适的替换零件](http://www3.telus.net/bc_triumph_registry/smoke.htm)，这就是夏普。

经过更仔细的检查，发现原来的主人用一根铜线绕过了电源保险丝，显然他们发现保险丝经常熔断，而不是试图解决问题，只是开枪打死了信使。

我们都可能在某个时候做过。在没有替换保险丝的情况下，我们可能会猜测接受电流所需的单股导线的数量，或者使用一条薄箔条包裹保险丝体。我们都会嘲笑那个关于[用扳手或者实弹当导火索](https://www.flickr.com/photos/dvanzuijlekom/10474490106/)的迷因。

因此，如果这个故事有什么寓意的话，那就是总是假设其他人都像你一样有能力做这种危险的修复，并且总是检查保险丝。
