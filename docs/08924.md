# 星期五黑客聊天:模拟模拟

> 原文:[https://hack aday . com/2018/03/28/Friday-hack-chat-simulating-analog/](https://hackaday.com/2018/03/28/friday-hack-chat-simulating-analog/)

对于任何硬件开发人员来说，模拟都是一个有价值的工具。您可以模拟一个微控制器，用一些 Javascript 让您的灯闪烁，而不是为一个长时间的调试会话构建硬件。不用为你的健身穿戴设备开发蓝牙协议，你只需快速编写一些 Javascript 代码，就能让它以这种方式工作。一旦你所有的 Javascript 都准备好了，你就可以转移到硬件上了。它节省了开发时间，也节省了资金。

[![](../Images/99239ecd40df6f177c9c4e6c6f0381bf.png)](https://hackaday.com/wp-content/uploads/2018/03/pspicehackchat_v01.jpg) 但这都是数字。如果你在模拟系统上工作，你会怎么做？幸运的是，有一个为模拟和混合信号分析而构建的系统，它已经存在了几十年。本周我们将讨论 PSpice，这是一款用于模拟分析的模拟器，可以给出原理图中每个节点的电压和电流。

[在本周的 Hack Chat](https://hackaday.io/event/96006-pspice-hack-chat-with-cadence) 中，我们将与[Abha Jain]和[Alok Tripathi]讨论 PSpice。[Abha]在 Cadence 工作了 19 年，过去十年一直是 PSpice 研发团队的一员。她是 VLSI 设计工具和技术的 MTech，拥有多项 EDA 专利。[Alok]1993 年毕业，获得电子工程学士学位。他于 1993 年开始在原子能部工作，担任粒子加速器的电源和控制系统设计师。目前，他在 Cadence 工作，是 PSpice 和 OrCAD 的产品工程架构师。

在这次聊天中，我们将讨论系统级仿真的挑战，提高电路设计的可靠性、成品率和生产率，Spice 仿真的问题，并回答以下问题:“在一个 1 欧姆电阻的无限网格上，相距一步之遥的两个节点之间的电阻是多少？”当然，我们鼓励你在 Hack Chat 中添加自己的问题。你可以通过[将问题作为评论留在这个 Hack Chat 的活动页面](https://hackaday.io/event/96006-pspice-hack-chat-with-cadence)上。
