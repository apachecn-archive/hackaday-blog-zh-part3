# 煎饼只读存储器？

> 原文:[https://hack aday . com/2017/02/21/pancake-rom-eat-only-memory/](https://hackaday.com/2017/02/21/pancake-rom-eat-only-memory/)

您可以将二进制编码的任意数据存储为 0 和 1 的模式。如何得到这些 0 和 1 取决于你自己。如果你有一种特别奇怪的心情，你甚至可以把它们储存成瑞典煎饼上的巧克力条。

奇怪的是，煎饼作为数字存储介质的可能性最初促使[迈克尔·科恩]在 2013 年开展了类似的项目，他在一个纸轮上编码了自己的名字。也许是明智的，他在一个更简单的媒介上做了原型。四年后，随着这项技术的完善，是时候升级改良瑞典煎饼技术(MSPT)了。

[![pancake_rom_bottom](../Images/1dfa4795a30a72622d18a6fc4e8ee823.png)](https://hackaday.com/wp-content/uploads/2017/02/pancake_rom_bottom.jpeg) 打造的亮点包括试图优化巧克力和煎饼之间的亮度差异。减少食谱中的糖量自然有助于通过减少焦糖化来增加对比。放置在旋转纸板平台下的棉球可以帮助稳定旋转的早餐/存储产品。

即使如此，[Michael]报告说，它需要多次尝试才能获得 16 个字节(bites？)的成功。数据被印在煎饼上，在我们看来非常清晰。对于反射传感器的更好的边缘检测，改进似乎是更大的问题。

数码电子产品和煎饼有什么关系？老实说，我们数不清这些年来出现过多少制作煎饼的机器人。这说明了什么？存储介质生产端的自动化！如果你能打印出一个[无限磁带的数据煎饼](http://hackaday.com/2016/03/11/endless-pancakes/)，你能做出一个[图灵机](http://hackaday.com/2016/07/12/a-very-modern-turing-machine-build/)吗？如果你决定回答这个问题，请告诉我们！
