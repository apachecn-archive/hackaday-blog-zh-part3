# 本周失败:不要把这些串行线绑得太高

> 原文：<https://hackaday.com/2016/04/25/fail-of-the-week-dont-tie-those-serial-lines-high/>

《一周失败》是 Hackaday 的一个长期系列节目。这些年来，我们经历了一连串有趣的、有启发性的，有时甚至是彻头彻尾的悲伤的错误，这些错误来自技术和制造商世界的各个角落。

你可能认为我们黑客作家只是记录别人的失败，嘲笑别人的不幸，就像学校里那个讨厌的孩子一样。但是不，我们和其他人一样容易失败，我们吃自己的狗粮，当灾难发生时，告诉世界我们的可耻灾难，这才是公平的。

所以我们来到了我的一周。我为我的合同客户自动化了一个测试过程。一些输出驱动一些继电器，一些输入来自按钮和微动开关。拿起 Arduino Uno 和原型盾，将右边的 14 条数字 I/O 线分成 7 个输出和 7 个输入。将 7 至 13 路由到 ULN2003 以驱动我的继电器，用一个 SIL 电阻组将 0 至 6 连接到高电平，这样我就可以用接地开关触发它们。工作完成了，这实际上是测试平台最终使用的硬件。

所以去 Arduino IDE 写我的草图吧。不涉及火箭科学，一组相当简单的输入、输出和定时器。上传到 Arduino，13 号针上的 LED 按预期闪烁。作为一名成功且有能力的工程师，去吃一顿当之无愧的午餐吧，他可以很快搭建起一个测试平台。

回到长椅上，吃着最好的英国酒吧食物，我启动了电脑，将屏蔽板插入 Arduino，并接通电源。我的素描成功了。但是等等！有轻微的 bug！回到 IDE，改一两行，上传草图。

我的失败来了。草图无法上传，IDE 报告了一个 COM 端口错误。“该死的 Windows 10 对 USB 串行端口的处理”，我想，因为我不是自己机器上的习惯性 Windows 用户。然后做了一些我很久没做的事情；潜入 Windows 控制面板来解决问题。因为它*有*是 Windows 的问题，对吗？

你们中经验丰富的阿杜因蒂人可能在四段前就发现了我的失败。我们都知道 Arduino 上的[引脚 0 和 1 是与串口](https://www.arduino.cc/en/Reference/Serial)共享的，但是谁会再想一想呢？我想我总是很幸运地从没有强制逻辑状态的线路上驱动这些引脚，并且从来没有把它们绑高。让它们保持逻辑 1，Arduino 就不能做串行的事情，所以草图就牢牢地留在 IDE 中。

每次我想上传一张新的草图时，我都可以把这个屏蔽去掉，但是因为我不需要所有这些输入，我只需要把那些引脚连接起来，然后把其他的输入转移到线上。那天晚上回家时，他是一名能力稍逊一筹的工程师，他迅速组装测试设备的能力有点受损。哼哼，至少修改后的草图起作用了，测试平台完全按照它应该的那样工作了。

这就是我本周的失败。你呢？
