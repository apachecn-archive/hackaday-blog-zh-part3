# 星期五黑客聊天:你如何与硬件合作？

> 原文:[https://hack aday . com/2018/02/07/Friday-hack-chat-how-do-you-collaborate-with-hardware/](https://hackaday.com/2018/02/07/friday-hack-chat-how-do-you-collaborate-with-hardware/)

开源软件的世界建立在协作的基础上。在世界的一个角落，有人可以修复一个软件中的 bug，并将其推送到 gits。在世界的另一个地方，其他人可以把这个补丁放到下一个版本中，很快每个人都有了更新更好的软件。互联网，或者说快速传输文本和二进制文件的能力，使这一切成为可能。

[![](../Images/28e188db534b0a870de1a8c34548a8dd.png)](https://hackaday.com/wp-content/uploads/2018/02/collaboratehardware.png) 硬件是另一个故事。有一个进入的财务障碍。你不仅需要一个仪表和一个好的熨斗，你可能还需要示波器、逻辑分析仪和一堆其他昂贵的工具。你需要购买你的 BOM。如果你使用 PIC，购买好的编译器可能是个好主意。硬件又硬又贵，所有抱怨的软件开发者都不知道他们在说什么。在硬件上合作比把一些代码推到云端要困难得多。

在本周的黑客聊天中，我们将讨论硬件项目的合作。这是一个关于如何与物理对象进行协作的深度探讨，本周我们将向一些最优秀的人学习。

我们本周黑客聊天的嘉宾是 SparkFun 电子公司的 Pete Dokter 和 Toni Klopfenstein。皮特以前是 SparkFun 的工程总监，现在是 SparkFun 电子产品的品牌大使。他主持 Pete 视频系列的*节目，阐述各种工程原理，他非常需要一辆 silverburst 莱斯·保罗和一辆 Sunn Model t。Toni 目前是 SparkFun 的产品开发经理。她是开源硬件协会的成员，并且每年都参加开源硬件峰会。在空闲时间，她一年中有 50 周的时间在她的艺术和电子项目中寻找灰尘。*

在这次聊天中，我们将讨论什么是协作硬件项目，如何让分布式开发为您的团队工作，以及您与几名相隔千里的硬件工程师一起工作的限制。这是一个困难的问题，比软件工程师的分布式团队要困难得多，也是一个对所有人都有意义的讨论。
