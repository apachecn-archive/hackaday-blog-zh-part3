# 鲁比克的机器人速度如此之快，看起来就像矩阵中的一个小故障

> 原文：<https://hackaday.com/2018/03/12/rubiks-robot-so-fast-it-looks-like-a-glitch-in-the-matrix/>

从法拉利到 F-16，有些东西就是看起来快。这个魔方解决机器人不仅看起来快，而且很快:它在 380 毫秒内解决了一个标准的魔方。在下面的视频中眨眼，你会错过它——即使在高速下，我们也很难跟踪这个解决方案采取的移动次数。看起来大约 20 岁。

打破之前 637 毫秒的机器人纪录只是[本·卡茨]所做的一个非常酷的构建的锦上添花。他和他的合作者[Jared]组装了一个具有明显工业外观的机器人——铝挤压底盘，六个带有高精度光学编码器的扁平伺服电机，以及用于爆炸遏制的聚碳酸酯面板，这些面板在开发过程中被证明是方便的。电机必须进行修改，以允许编码器连接到后面，并定制电机控制器。[Jared]想出了一个独特的板来同步六个电机，并防止面之间的碰撞。机器视觉仅由两个 PlayStation Eye 摄像头提供；每台摄像机安装在围栏的对角，一次可以看到三张脸。他们在区分红色和橙色时遇到了一点麻烦，用一支记号笔就解决了。

[Ben]和[Jared]认为他们可以通过调整在这里或那里节省几毫秒，但即使如此，这也是优化和集成方面的一个很好的教训。我们之前已经介绍过魔方机器人，比如[这种双电机缓慢而稳定的设计](https://hackaday.com/2017/12/02/solving-a-rubiks-cube-with-just-two-motors/)和[这种六电机的构建](https://hackaday.com/2016/01/27/robot-solves-rubiks-cube-in-just-one-second/)，它可以在不到一秒的时间内解决一个立方体。
