# 无人机很棒，同时也很可笑

> 原文：<https://hackaday.com/2015/11/06/bitdrones-are-awesome-ridiculous-at-same-time/>

一开始我们觉得这很棒，后来我们觉得这很可笑，现在我们基本上确定了“可笑的我”。

Bitdrones 是一种人机交互的原型，它使用微型四轴飞行器作为 3D 沉浸式显示器的像素。那是超级酷的部分。“像素晶体管”顶部有一个 LED。“ShapeDrones”有一个薄薄的笼子，由彩色 led 照明，使它们成为真人大小的彩色体素。(爽！)最后，“DisplayDrone”安装了一个触摸屏。一台计算机跟踪每架无人机在房间中的位置，它们一起工作来创建一个步入式 3D“显示器”。到目前为止，一切都很好。

还有更好的。因为控制无人机的程序知道每架无人机的位置，它可以告诉你何时在太空中移动无人机。这非常酷，为新的互动打开了平台。DisplayDrone 就像一个微型飞行手机，所以你可以不用手就能和在你房间周围徘徊的朋友聊天。请看插播在广告下方的视频。

另一方面，对于人机界面，我们不得不说他们选择的例子并不合适。例如，看看视频中无人机被用来表示目录中的文件(42 秒)。要么是目录里只有三个文件，要么是无人机用光了。在这三个位置之间轮换有点慢，因为实体无人机必须飞到新的位置来选择文件。我们在想那个目录，里面有我们刚从相机里下载的 200 张照片。

视频中他们使用三个 ShapeDrones 来代替 3D 物理模型的部分(52 秒)再次令人惊叹。用户将无人机放置在空间中，它们在那里盘旋，并且可能在某处的计算机中的 3D 模型的某个部分周围移动。我们喜欢它。但是他们又一次碰到了虚拟和真实的区别；三个体素，即使它们非常优雅并且是 RGB 颜色的，也不足以用于任何严肃的建模。即使是一个普通的现实世界项目也需要一大群这样的东西，这将是非常棒的，但是非常昂贵。

最后，作为这个项目同时令人敬畏和挠头的缩影，他们将捏手势(放大或缩小图像)从触摸屏转换到现实世界(视频中的 1:52)。原则上这非常酷，我们热爱这项运动。但在视频中，演示者差点被第三个向内收缩的像素撞到肩膀。我们想象一个长头发的用户和一个有更多直升机的模特。恐怖啊！

最后，用交互设计术语来说，虚拟世界和现实世界有不同的[启示](https://en.wikipedia.org/wiki/Affordance)，这是项目试图模糊或忽略的物理事实。四轴飞行器不是像素:你不可能有一百万个这样的飞行器，即使你可以，它们的重新配置也很慢，而且更难操纵。也就是说，这里已经设计了一些美丽的东西，我们绝对相信有一个适合它的应用程序。

因此，对于导航文件，我们将坚持使用鼠标。但是我们真的想要一屋子会飞的 RGB 立方体！
