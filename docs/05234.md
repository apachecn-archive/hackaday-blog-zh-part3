# 通过着色使 Linux 更好

> 原文:[https://hack aday . com/2017/03/07/better-Linux-through-colouring/](https://hackaday.com/2017/03/07/better-linux-through-coloring/)

如今，网络安全是每个人都关心的问题。像摄像头这样的嵌入式设备已经被坏人用来在互联网上发起攻击。人们担心语音命令设备或家庭自动化系统的数据泄露。这也适用于我们构建和部署的自主系统。

许多网络感知系统在某个地方使用 Linux 一个很大的例子是几乎所有基于 Raspberry Pi 的项目。当您部署 Pi 时，您对安全性考虑多少？有一个适用于 Linux 的高级安全系统(包括您在 Pi 上使用的大多数版本)叫做 SELinux。前面增加的字母代表“安全增强”,这个项目最初是由美国国家安全局和 RedHat 发起的。说真的，RedHat 实际上有一本帮助解释一些基本概念的涂色书。

我们不太确定涂色书的格式是否真的是正确的方法，但它是一种轻松且有知识性的读物(尽管我们没有很好地保持一致)。我们的一个抱怨是，它实际上并没有向您展示任何东西，它只是解释了 SELinux 中不同类型的保护背后的思想。如果你真的想在 Pi 上设置它，Pi 网站上有一个页面[会有所帮助。如果你有一个小时的时间，你可以在下面的视频中很好地了解 SELinux 的使用。](https://www.raspberrypi.org/forums/viewtopic.php?f=66&t=16738)

默认情况下，Linux 安全模型非常简单。你有四个概念上的人群:根用户、某些东西的所有者、用户组，以及不符合任何上述类别的人。文件和看起来像文件的东西可以拥有适用于所有者、所有者的组和每个人的读、写和执行权限。对于目录和一些其他特性有一些特殊的规则，但简单来说就是这样。制作一个你能读、写和执行的文件(像一个 shell 脚本)是很容易的。可能你组里的用户可以读，可以执行，其他人只能执行。在一些 Pi 系统上，这可能是好的，因为无论如何你是唯一的用户，所以组和 all 没有多大意义。然而(正如你将在视频中看到的),使用它来控制访问，比如说，一个网络服务器，可以使你的系统不容易受到攻击。

使用 SELinux，您可以在进程和文件系统对象上贴标签，并使用它们来控制访问。涂色书中的例子着重于确保狗吃正确的狗粮，猫吃猫粮。使用这些方法可以获得非常细粒度的控制。你也可以像政府一样使用 MLS(多级安全),让事情成为秘密、绝密等。

花一些时间来确保您获得了基本的 Linux 安全性。这样，黑客(坏的那种，不是我们这种)将不得不更加努力地工作来造成混乱。我们已经看到了一些相当不可思议的努力，比如用 6502 代码黑掉一个现代的 Linux 系统，目标是任天堂的 T1。众所周知，[互联网连接设备](https://hackaday.com/2015/08/10/defcon-vs-iot-on-hackability-and-security/)正成为黑帽子们的首选目标。
