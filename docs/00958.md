# Rory Aronson 谈记录开源项目

> 原文:[https://hack aday . com/2015/12/16/Rory-a ronson-on-documentating-open-source-projects/](https://hackaday.com/2015/12/16/rory-aronson-on-documenting-open-source-projects/)

每个项目都是从一个想法开始的。有时候那些想法比一个人，甚至一小群人都要大。这就是罗里·阿伦森(Rory Aronson)在 2015 年 Hackaday 奖决赛参赛作品 Farmbot 中的处境。文档是[罗里]的关键。Farmbot 最初是以白皮书的形式问世的。该论文包括对合作者的请求，使得这从第 0 天起就是一个开源项目。文档在整个 Farmbot 项目中一直很重要，所以这自然是[Rory]在 2015 年[hack aday 超级大会](http://hackaday.io/superconference)上的话题。

Rory 的使命陈述是“伟大的文档是有趣的、彻底的、简洁的传播知识的对话”。他当然是对的。任何人都可以挑出可怕的文件。要么太长，要么太短，要么过时，要么就是完全错误。[Rory]努力保持文档简短，并抓住 Farmbot 项目的要点。他不是唯一一个致力于此的人——那是“对话”的部分。Farmbot 有一个论坛和一个社区驱动的文档站点，这使得文档很容易更新。

没有人喜欢写文档——这没有跳到下一个项目有趣。回报是激励人们和你一起工作，在你已经打下的基础上继续努力。当你在记录一个项目的时候，来点幽默通常会有所帮助。这就是乐趣所在。对[罗里]来说，乐趣体现在每一个组装步骤结束时，一些双关语、迷因图片和他标志性的竖起大拇指的照片。休息之后看看[Rory 的]视频，在评论中给我们一些你的文档提示！
