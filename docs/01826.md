# 反黑客:免费自动 SSL 证书

> 原文:[https://hack aday . com/2016/03/20/anti-hack-free-automated-SSL-certificates/](https://hackaday.com/2016/03/20/anti-hack-free-automated-ssl-certificates/)

你想把你的信用卡号码放到一个网站上。你知道去找一个安全的网站。但这能证明什么呢？而且现在这么多电子项目都有 web 服务器(好吧，我就说吧……物联网)，你还需要保护你的 Web 服务器吗？

曾几何时，获得一个安全证书(至少是一个有意义的证书)要花一大笔钱。然而，一些主要参与者(如思科、谷歌、Mozilla 和许多其他公司)支持的一项新倡议希望给你一个免费的 SSL 证书。他们能够负担得起的一个原因是，他们已经实现了验证过程的自动化，因此提供证书的成本非常低。

事实并非总是如此。最初，可信证书相当昂贵。要理解为什么，您需要考虑 SSL 证书的真正含义。首先，您可以通过简单地创建一个证书来获得一个免费的证书。价格是合适的，但结果还不尽如人意。

证书包含服务器的公钥，因此任何密钥都足以加密发送给服务器的数据，这样其他人就无法窃听。它不能证明服务器就是他们所说的那个人。一个自生成证书说“嘿！我是你的银行！”但是没有证据证明。

为了得到这个证据，你需要两样东西。您需要由证书颁发机构(CA)签署的证书。您还需要 Web 浏览器(或其他客户端)来接受 CA。一个精明的用户可能会安装特殊的证书，但是对于大多数实用程序来说，您需要一个浏览器已经识别的 CA。实际上，事情要比这复杂一点。您的 CA 可能被另一个 CA 识别。该 CA 可能被另一个 CA 识别。这可以持续一段时间。浏览器只需信任链中的一个 ca。

这意味着 CA 有效地证明你就是你所说的那个人。在过去，这是一个费力的过程，需要 CA 证明你的 URL、公司名称、地址和其他细节。如今，大多数证书只验证 URL。是否转到正确的 URL 取决于您。

然而，这使得 CA 验证更加简单。使用“让我们加密”验证，您必须有权配置 DNS 记录或将文件放在服务器上，这是网站管理员已经熟悉的过程。自动化过程将验证您确实进行了更改，从而证明您拥有该域。

让我们加密将很快成为电子自由基金会的一部分，并将获得一个新的名字。然而，他们已经颁发了超过一百万张证书。大多数浏览器和操作系统都可以使用这些证书，尽管 Windows XP 有一些限制。

整个 SSL 系统通常运行良好，但并不完美。在之前，我们已经讨论过 Google 的一些[证书安全问题。](http://hackaday.com/2013/12/09/google-security-certificates-forged/)

随着 Web 服务器在电子项目中变得越来越普遍，一个免费的可信证书源将会派上用场。你可以在下面观看安装过程的视频。
