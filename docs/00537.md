# 谁说 FORTRAN 死了？

> 原文:[https://hackaday.com/2015/11/01/who-said-fortran-is-dead/](https://hackaday.com/2015/11/01/who-said-fortran-is-dead/)

美国宇航局迫切需要一个 FORTRAN 开发者来支持旅行者号飞船。*大众机械师*采访了航海家项目经理【苏珊·多德】，他希望在退休后接替【拉里·佐塔雷尔】的职位。

我们有很多人评论我最近的 Hackaday 文章，[“这不是你父亲的 FORTRAN”](http://hackaday.com/2015/10/26/this-is-not-your-fathers-fortran/)，他们在某个时候学习了这门语言。也许你们中有人想申请？你需要尽快这么做！NASA 希望给新员工六到十二个月的在职培训。你也需要温习一下你的老式汇编语言。

[![light data system hw](../Images/62689df2201ea8195975018070c2cf97.png)](https://hackaday.com/wp-content/uploads/2015/10/light-data-system-hw.jpg) 这两位航海家是美国国家航空航天局第一批[使用计算机的航天器](http://history.nasa.gov/computers/Ch6-2.html)。资源仅限于每个探测器上发现的三台 40 岁的计算机。他们操作飞船的科学和飞行软件。该软件是最近才更新的，仅在 25 年前的 1990 年更新过。

一个大问题是很多工程设计资料已经不存在了。人们对很久以前发生的事情和做出决定的原因的记忆有点模糊。但是当出现问题时，NASA 有一份这些前工程师的紧急名单。这意味着这可能不仅仅是一份为古代硬件编程的工作，你可以找到很多理由与这个领域的先驱进行互动！

这将是一个可怕的黑客。有人能在 120 亿英里外进行远程计算吗？
