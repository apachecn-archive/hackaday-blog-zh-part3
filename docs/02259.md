# 树莓派气球飞得太高，爆炸，但幸存下来

> 原文：<https://hackaday.com/2016/05/07/raspberry-pi-balloon-goes-too-high-goes-boom-but-survives/>

有些人喜欢在星期三下午兴奋起来。布莱克·梅萨实验室的凯文·哈伯德喜欢爽到极点。甚至比预期的还要高:上个月，他驾驶一个由树莓皮驱动的氦气球飞到了 103，000 英尺的高空。它本来应该达到 90，000，但控制器代码中的一个错误意味着它会飞得更高，爆炸并坠落到地面。这都要归功于他的代码中多了一个散列符号。

[Kevin]是一个名为气球工程师的团队的成员，他们正在参加全球太空气球挑战赛，这是一个从多个地点同时放飞气球的项目。Balloongineers 的参赛作品被称为 HAB1，围绕一个 Raspberry Pi、一个 FPGA 看门狗系统、一个 uBlox GPS 和一个铱星调制解调器构建。他们的想法是，他们的气球将上升到 90，000 英尺，在那里它将释放一些氦气，悬停，并随着盛行风向东移动。通过卫星发回报道。不幸的是，出了点问题。气球没有正确报告，并继续上升，最终达到 100，000 英尺以上，在那里破裂并坠落到地球。

[Kevin]以为一切都完了，包括 HAB1 使用的昂贵的卫星调制解调器。但是第二天，他的气球给他发来了一封邮件，报告说它在 300 英尺的高空仍然很健康。恢复后，他分析了 HAB1，并找出了发生的事情:当系统试图打开通风口释放气体时，一个错误的散列符号导致系统锁定。

拯救钻机的是[凯文]在建造它时的远见。虽然系统没有完全按计划工作，但 FPGA 看门狗(昵称为蜥蜴脑)最终注意到主计算机被锁定，并重新启动它，使系统能够向[Kevin]报告。额外的恢复模式每小时唤醒系统一次，发送一个位置并返回睡眠，这允许小电池继续为系统供电，直到第二天早上收到信号。这是系统中的一个聪明设计，允许他们恢复硬件，即使在 90，000 英尺悬停的主要目标没有实现。

[Kevin]用一些任何工程师都应该明智考虑的注释结束了他对这次飞行的记录。其中最主要的是他决定让系统很少检查卫星连接上的手动覆盖。他决定这样做是为了省钱:每次在铱星上查看信息都要花费一定的费用。如果他更频繁地检查，他可能会通过手动排放氦气来更早地解决问题，从而更好地控制下降和回收。在任何项目中，只有当你能找出问题所在，收集尽可能多的信息来帮助你下次避免失败时，失败才有用。
