# 语音用户界面设计实践

> 原文：<https://hackaday.com/2018/02/08/voice-user-interface-design-practices/>

网站以前比现在丑。当然，你仍然可以找到一些灾难，但是回到网络的早期，你会发现闪烁的横幅、旋转的文本、音乐背景和奇怪的导航主题。实践在发展，现在专业设计的网站之间的差异已经很小了。

在超文本世界的一面镜子中，同样的事情也将发生在语音用户界面(或 VUIs)上。随着 Google Home 和亚马逊 Echo 等产品获得更多用户，开发 VUIs 将成为一件大事。我们也开始看到使用 vui 的黑客项目，要么利用大家伙，在 Raspberry Pi 上使用本地代码，甚至使用专用的语音硬件。那么，VUI 的最佳实践是什么呢？[弗雷德里克·古森斯]在最近的一篇文章中分享了他对这个主题的想法。

事实上，许多设计过程[Frederik]建议模仿传统的用户界面设计来定义用例并规划流程。然而，围绕可用的语音交互有一些独特的问题。

总之，VUI 的要点是:

*   简单和对话
*   确认完成
*   错误策略
*   考虑额外的安全层

您可以在原始帖子中找到更多细节，其中也包括一些关于工具和设计的一般建议。

如果你正在寻找一些更详细的东西，资深语音开发人员[Jo Jaquinta]在评论中发布了一个视频，关于他在都柏林就同一主题发表的演讲，你可以在下面看到。同时，如果你想尝试自己的声音发展，我们已经看到这可能比你想象的要容易。如果你想在本地完成所有工作，即使是[Arduino 也能听到你要说的话](https://hackaday.com/2016/12/11/arduino-clock-is-hal-1000)。
