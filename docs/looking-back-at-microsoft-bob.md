# 回望微软鲍勃

> 原文：<https://hackaday.com/2018/01/29/looking-back-at-microsoft-bob/>

每个行业都至少有一个。汽车拥有埃德塞尔。PC 硬件有 IBM PCJr 和微通道总线。在软件世界里，有鲍勃。如果你不记得他，鲍勃是微软 1995 年对为什么电脑如此难用的回答。[LGR]让我们怀旧地回顾了鲍勃，并得出结论说我们几乎不认识他。

鲍勃把你的桌面变成了房子而不是桌子。他也有帮手，包括臭名昭著的会说话的回形针，在鲍勃被埋葬后很久，它在微软办公室里遭到了攻击。

微软为鲍勃制定了宏伟的计划。有一本杂志和附加软件(显然只有一个标题发布)。当然，如果你想自己安装 Bob，你需要启动 Windows 3.1——记住，这是 1995 年。

要登录，你必须敲开红色的大门，然后告诉这只乐于助人的狗你所有的个人信息。每个用户都有一个私人房间，所有用户将共享其他房间。

我们喜欢展示我们年轻时代伟大的老式计算机的逆向计算。这是一个反例。鲍勃是个大失败者。《个人电脑世界》将其评为有史以来 25 款最差科技产品的第 7 名，CNet 称其为十年来最差产品的第一名。

一旦你受够了 1995 年失败的软件，你总是可以[阅读](https://hackaday.com/2014/02/17/a-pick-and-mix-fpga-retrocomputer/)一些更成功的 [Z80 克隆](https://hackaday.com/2016/09/03/talking-diy-z-80-retrocomputer-complete-with-dev-tools/)。或者你可以回到过去，看看[20 世纪 60 年代和 70 年代的用户界面是什么样子的](https://hackaday.com/2017/08/10/getting-started-with-blinking-lights-on-old-iron/)。
