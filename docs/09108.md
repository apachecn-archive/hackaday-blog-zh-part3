# 被遗忘的工作站:Sun JavaStation

> 原文：<https://hackaday.com/2018/04/10/the-forgotten-workstation-sun-javastation/>

这些天，关于 Java 的讨论往往围绕着甲骨文和谷歌在法庭上一决雌雄。但早在 1996 年，孙还是 Java 的守护者，并大力推广 Java。他们甚至发布了只运行 Java 小程序的无盘工作站。Sun JavaStation 被亲切地称为“咖啡先生”，[卡梅伦·格雷]想向你展示它是如何工作的，以及它的内部是什么。

一颗螺丝就能把内部零件从小盒子里取出来。内部看起来像是那个时代的密集 PC，尽管内部的部件与典型的 PC 相去甚远。CPU 是一个直接焊接在主板上的 110 MHz microSPARC II。四个 RAM 插槽可以占用高达 64 兆字节的 PC RAM。

与 Sun 成熟的工作站不同，JavaStation 采用了标准的 PC 外设。例如，显示器连接是标准 VGA，并且有正常的 PS/2 键盘和鼠标端口。这个盒子的驱动力是价格，因此拥有廉价的商品 I/O 是一个卖点。

[卡梅伦的] JavaStation 已经丢失了它的配置，所以他不得不使用串行终端来重置一些关键参数。之后，机器能够通过其网络连接自行启动。毫无疑问，操作系统是 JavaOS。

顺便说一句，如果你碰巧有一台这样的机器，它们[可以运行 Linux](http://www.ibiblio.org/pub/Linux/docs/HOWTO/other-formats/html_single/JavaStation-HOWTO.html) 。原来运行 Linux 使机器比原来的 JavaOS 更快，并且提供了更好的软件选择。例如，你可以看到 Corel Office 在机器上运行，[Cameron]说应用程序和性能的缓慢加载是用户的主要抱怨。

这些都被 Sun Ray 瘦客户端所取代，我们已经看到它被转化成了一台树莓派电脑。当然，如果你只是想要一个瘦客户端，你[不需要去复古做那个](https://hackaday.com/2015/08/16/new-version-of-raspberry-pi-thin-client/)。
