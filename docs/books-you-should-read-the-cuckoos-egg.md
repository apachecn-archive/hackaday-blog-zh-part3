# 你应该读的书:《布谷鸟的蛋》

> 原文：<https://hackaday.com/2017/10/19/books-you-should-read-the-cuckoos-egg/>

20 世纪 80 年代中期是剧烈变革的时期。在美国，里根时代即将结束，冷战正在升温，而 IBM 个人电脑是最新的新事物。将世界各地较大的大学研究中心连接在一起的相对较少的电线为一种新的心跳——互联网协议(IP)——而跳动，虽然万维网仍然是十年左右的事情，但对于越来越多精通计算机的探险家和冒险家来说，互联网是一个真实的地方，他们准备在虚拟的海洋上启航，探索和利用这一新的前沿。

1986 年，天文学家克利夫·斯托尔最近失去了他的研究经费，他被劳伦斯柏克莱实验室物理系的管理层一挥手任命为计算机系统管理员。斯托尔受命前去管理，开始了他第一天工作时看似简单的任务:调查电脑账户计时收费中的 75 美分错误。他一点也不知道，这六比特的过度收费将在接下来的六个月里占据他的生活，并让这个自称为伯克利嬉皮士的人与联邦调查局、中央情报局、国家安全局和德国联邦安全局(Bundeskriminalamt)密切合作，所有这些都是为了追寻源头:一个黑帽黑客的巢穴和一个错综复杂的国际间谍网。

出版于 1989 年，在它描述的事件发生后不久，*布谷鸟的蛋:通过计算机间谍活动的迷宫追踪一个间谍*对于一本表面上是关于技术的书来说是一个奇怪的野兽。这本书读起来很像一部小说，大量对话来自斯托尔在调查期间做的大量笔记。他承认有些对话没有逐字记录，这对于任何听过斯托尔讲 T4 的人来说都是可以理解的；至少可以说，准确跟踪如此巨大的狂躁能量是很困难的。但他对细节的关注令人印象深刻，比如当他在电话会议上反复列举端口和线路分配，试图追踪他的黑客用来侵入 LBL 电脑的全国电话网络和网络连接时。

虽然[写了一篇更正式的论文](http://pdf.textfiles.com/academics/wilyhacker.pdf) (PDF 链接)，*的《布谷鸟的蛋》*更吸引人，因为读起来像一个故事，而不是干巴巴的技术叙述。这本书刚出版时，我拿起了它，可能是为了逃避我的本科学习，逃到一个不同的、比分子生物学有趣得多的世界。当我读斯托尔的书时，我立刻感觉到了他的亲切——我们都是不适合的科学家，都试图找到一种方法将我们对计算机科学的兴趣与我们选择的领域结合起来。像他一样，我觉得我的选择可能有点不明智，但读了他的事迹后，我发现外面有更广阔的世界。

不少黑客读者会记得互联网的早期，尽管《布谷鸟的蛋》具有叙事性质，但仍有大量过去的科技花絮和爆料来满足对细节的需求。这个标题来源于斯托尔的黑客用来提升自己在 LBL Unix 机器上的超级用户权限的方法；利用 Emacs 中一个当时未知的漏洞，黑客能够将一个被黑版本的 atrun 程序转移到系统空间；它将在五分钟内运行，并给予他超级用户权限，就像布谷鸟在另一个物种的巢中下蛋，让她的小鸡孵化和成长，对宿主家庭造成损害。

在我看来，这本书最精彩的部分是当科学与工程发生碰撞的时候。在一次与诺贝尔奖获得者 Luis Alvarez 在餐厅的偶遇后，斯托尔开始考虑用科学术语来描述对黑客的追捕。Luis Alvarez 因发现暗示 6500 万年前小行星撞击导致恐龙灭绝的铱边界而闻名。那就是提出一个假设，设计一个实验，检验你的假设。追踪并最终收集到足够的证据来逮捕不止一个黑客，而是一个向克格勃出售机密的组织，这一切之所以成为可能，完全是因为斯托尔细致的观察和科学方法的应用。斯托尔的工作是这种方法成为我自己职业生涯蓝图的原因，即使我并不总是坚持这个计划。

这本书还预示了安全国家的崛起，并表明我们这些天在美国已经习惯的一切，包括美国国家安全局的国内监控，可能在 80 年代中期被积极地，如果秘密地追求；一名 NSA 分析师声称国内监控会导致监禁，这听起来有些空洞。

我认为《布谷鸟的蛋》是任何对科学和工程之间的接口感兴趣的人的必读之作，到目前为止，它甚至已经成为了科技文学的经典。这本书非常值得一读，但当然如果你宁愿花一个小时看一集 NOVA，休息后的视频会给你提供要点，并给斯托尔书中的名字加上面孔，因为许多参与的人都在重演中使用。但是公平的警告:如果你跳过这本书，你会错过斯托尔的搜索和计算机历史上最著名的蠕虫攻击之一[之间的联系。](https://en.wikipedia.org/wiki/Morris_worm)
