# Hackaday 奖参赛作品:FabDoc 是项目图像的版本控制

> 原文：<https://hackaday.com/2017/08/11/hackaday-prize-entry-fabdoc-is-version-control-for-project-images/>

FabDoc 是一个有趣的概念，试图解决一个我们很多人都没有意识到的问题。有大量的软件版本控制系统，但许多项目也有硬件元素或组装过程。那些物理元素需要被文档化，但是这个过程不容易适应那些使软件开发和协作更容易的工具。[郑嘉颖]将 FabDoc 总结为“一个捕捉延时照片作为预提交的系统。”

有了 FabDoc，相机会自动记录物理开发过程，使开发人员能够专注于工作并在以后进行审查。来自摄像机的图像被视为预提交。在审查时，开发人员选择相关的关键映像(忽略死胡同或错误的开始)并提交它们。它是开发过程物理部分的版本控制和提交系统。目标是消除为了拍照而停止工作过程的负担，自动记录开发过程并将其附加到特定项目，并允许轻松管理提交哪些图像。

目前的系统使用的是一个树莓 Pi Zero，在安全眼镜上安装了一个摄像头，还有一些支持软件。一些想法当然是为了使系统尽可能易于使用和管理；建立存储库后，扫描二维码会告诉系统做什么和把它放在哪里。我们的目标是让 FabDoc 快速易用，这样它就可以简单地在无人值守的情况下工作。

前一段时间，我们看到了版本控制的视觉扭曲，PCB 的视觉差异，这是一个很好的想法，以不同的风格来可视化地表示 PCB 设计之间的变化。看到有人尝试改进容易被认为是理所当然的过程总是令人兴奋的。
