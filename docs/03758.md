# 从来没有两次相同的颜色:为什么 NTSC 是如此怪异

> 原文：<https://hackaday.com/2016/10/06/never-twice-the-same-color-why-ntsc-is-so-weird/>

有没有想过，从技术角度来看，为什么北美的模拟电视如此怪异？[standupmaths]做了，所以他稍微研究了一下普遍讨厌的彩色电视 NTSC 标准的历史，结果不仅解释了美国电视标准是如何形成的，也给工程师们上了一课，他们有时不得不做出不雅的设计妥协。

在我们在评论中陷入 NTSC 和 PAL 之间的激烈争论之前，作为美国居民，我们必须承认我们的模拟彩色电视标准很糟糕。但是随着[站立的马茨]的深入探索，这种疯狂是有方法的。他的主要抱怨集中在国家电视系统委员会决定使用 29.97 fps 的帧速率，而不是更合理的(对于 60 Hz 交流电网)30 fps。我们将把细节留到下面的视频中，但可以说，像许多设计决策一样，这个决策必须让多个用户满意。或者至少同样悲惨。最后,[standupmaths]很容易看出为什么最不糟糕的决定是从 30 fps 稍微降低刷新速度。

考虑到他们工作的限制，NTSC 工作得如此之好的事实是非常令人印象深刻的，是一个相当史诗般的黑客。显然也很鼓舞人心。我们最近在这里看到了相当多的模拟电视帖子，比如使用 SDR 来传输 PAL 信号或来自微控制器的 NTSC 信号。
