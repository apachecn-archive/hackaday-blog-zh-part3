# 《彗星美人》

> 原文:[https://hackaday.com/2016/11/05/all-about-eve/](https://hackaday.com/2016/11/05/all-about-eve/)

今天的大多数编程语言看起来都非常相似。当然，也有一些小的区别(Python 使用空格，Ruby 和 Perl 有一些奇怪的结构)。然而，在 20 世纪 60 年代和 70 年代，许多编程语言都相当神秘。Algol、APL 和 LISP 是不寻常的编程语言的很好的例子。甚至 FORTRAN 和 PL/1 也很难读懂。RPG 和 COBOL 试图让编程变得更容易理解，尽管你可能会说它们都没有统治世界。如今，大多数编程语言与 FORTRAN 的相似性都超过了这两种语言中的任何一种。

一种新的编程语言 [Eve](http://www.witheve.com/) ，声称是基于多年来从人类角度而不是从计算机角度对编程的研究。结果是一种通过模式匹配而不是通常的控制流工作的语言。它还存在于可以作为文档的 Markdown 文档中。你可以看到一个关于夏娃的视频，在下面。

这两个都不是全新的想法。斯诺鲍、AWK 和普洛斯都涉及一些模式匹配。早在 20 世纪 80 年代，唐纳德·克努特就在推广识字编程。然而，Eve 理解像 web 浏览器这样的现代结构。

你可以在网上探索一下 Eve。不要忽略页面右上角的按钮。他们允许你运行 Eve 代码，甚至检查它。只需确保每个代码片段右上角的复选框都有一个复选标记。或者，单击左上角的示例文本，通读一些示例，例如“Flappy Eve”。

今年早些时候，我们已经讨论过文化编程。如果你想要 Eve 的反面，也许 [Rust](https://hackaday.com/2015/12/18/programming-with-rust/) 会适合你。
