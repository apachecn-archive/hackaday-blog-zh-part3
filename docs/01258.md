# 基本上，它是《我的世界》

> 原文:[https://hackaday.com/2016/01/19/basically-its-minecraft/](https://hackaday.com/2016/01/19/basically-its-minecraft/)

真的喜欢《我的世界》。你怎么知道？快速浏览一下他的 YouTube 频道应该会让你信服，尤其是他在《我的世界》构建了一个成熟的 BASIC 解释器的那个。正如你所料，它不会赢得任何速度比赛，但它确实有效。

对于新奇和令人惊叹的因素，这是惊人的。实际上，很难想象它的真正价值，因为一个新程序员有很多方法可以接触到 BASIC。尽管如此，你还是不得不佩服做出这种尝试的勇气。一个 Hackaday 的发帖者(他将保持匿名)曾经赢得了一箱啤酒，打赌他或她能用 BASIC 编写一个 BASIC 编译器，所以我们不拘泥于实用性。

因为解释器太慢了，我们希望演示视频中的质数算法(见下文)能更高效一点。首先，您不必检查大于目标数平方根的数，因为该范围内的任何因子都会显示另一个大于平方根的因子。如果 BASIC 解释器没有平方根函数，你至少可以停在目标数的一半。当然，你也可以保留一个质数表，跳过测试任何非质数的因素。毕竟，如果(比如)15 平分你的数，那么 5 和 3 也会平分，对吧？但那是算法设计，并没有从基础中拿走任何东西。

我们已经看到 [a 6502 在《我的世界》](http://hackaday.com/2012/05/20/building-a-6502-in-minecraft/)跑来跑去，所以很高兴看到 BASIC 也有出头之日。我们仍然看到一些基本利益，比如相当新的[解释器将一枚 ESP8266](http://hackaday.com/2015/08/29/basically-its-an-esp8266/) 变成了一枚仿制的[基本邮票](http://hackaday.com/2015/08/27/before-arduino-there-was-basic-stamp-a-classic-teardown/)。
