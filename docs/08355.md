# 在纸上运行程序

> 原文：<https://hackaday.com/2018/01/22/running-programs-on-paper/>

一个简单的事实是，大多数为个人电脑设计的程序都包含相同的交互方式，几乎与目的无关。文字处理器、图形工具，甚至游戏——绝大多数交互都是通过键盘和鼠标进行的。然而，有时试验替代技术让用户与代码交互会很有趣——[纸质程序就是这样一种令人兴奋的方式。](https://paperprograms.org/)

Paper Programs 是各种现有技术的组合，旨在创建一种与高度有形的代码进行交互的方式。该设置包括一个投影仪和一个可以看到投影区域的网络摄像头，以及在浏览器中运行的 Javascript 程序。程序可以在浏览器中编辑，然后用特殊的彩色圆点打印出来。当页面被放置在投影区域时，这些点识别独特的程序，并被网络摄像头拾取，服务器执行相关代码，投影回页面。

这是一个系统，它创造了一种非常有触觉的与程序交互的方式——通过移动页面或将不同的页面相邻放置，程序可以以各种方式进行交互。该系统还支持协作，允许用户直接在浏览器中编辑代码。

这个项目让我们想起了早期在 DIY 多点触摸屏上的作品，但是更关注与底层代码的直接接触。还有哪些与代码交互的独特方式？请在评论中告诉我们。
