# 破解 Excel 进行 3D 渲染

> 原文:[https://hack aday . com/2018/02/18/hack-excel-for-3d-rendering/](https://hackaday.com/2018/02/18/hack-excel-for-3d-rendering/)

[C Bel]教 Excel，他有一个问题。我们大多数人——尤其是我们这些黑客——会立即编写一个 VBA (Visual Basic for Applications)宏来处理 Excel 中的棘手问题。根据[C Bel]的说法，这不仅对非技术用户来说很困难，而且效率也不高。为了证明并不总是需要 VBA 宏，他只用 Excel 公式编写了一个 [3D 游戏引擎。他确实不得不求助于 VBA 来获取用户输入，在极少数情况下，还要提高大型算法的性能。你可以在下面的视频里看到他的结果或者下载下来自己试试。](https://www.gamasutra.com/blogs/CBel/20180213/308549/3D_engine_entirely_made_of_MS_Excel_formulae__Enjoy_this_Doomxls_file_.php)

这个游戏有点像末日。有点吧。正如你所料，它的速度并不快，敌人是一个红色的大斑点，但正如一句古老的俄罗斯谚语所说，“神奇的不是熊跳得好，而是熊会跳舞。”(感谢[肖恩·博伊斯]回忆起这句话。)

如果你想尝试一下或者深入了解内部情况，你可以下载带有或不带有 VBA 的电子表格。这确实令人印象深刻，包括:

*   一个无限程序化生成的迷宫地图
*   实时光线跟踪
*   遮挡计算
*   基本照明渲染
*   照明和计算着色器
*   自然排量发动机

我们同意这比宏更好吗？我们不确定，但我们印象深刻的是你可以通过操纵公式来实现这一切。

这并不是说我们自己没有滥用电子表格(看看电子表格中的 [CARDIAC 和——虽然不是我们的——小人](https://hackaday.com/2017/10/13/computers-that-never-were/))。我们甚至用它们来进行[电路分析](https://hackaday.com/2016/08/31/circuit-design-spread-the-joy/)。
