# 诺亚·费恩和创造者的思想

> 原文：<https://hackaday.com/2015/12/09/noah-feehan-and-the-mind-of-the-maker/>

太多时候，我们发现自己在这些页面上展示项目，却没有给予项目制作者太多的思考。然而，在 LED 面板、github 页面和 PCB 的背后，有一个人让硬件奇迹发生。不仅仅是一个人，而是整个文化中的人们，因为软件错误和焊接不良而让他们的意识时间流血到深夜。Noah Feehan 是这些老兵中的一员，在今年的 T2 超级黑客大会上，他接触了这种文化。诺亚带来的不是项目，而是设计技巧和富有感染力的热情，这将让你重新思考如何在 DIY 的土地上利用你的时间和空间。凭借在艺术和工程设计方面的十年经验，Noah 向其他黑客提供了他最好的技巧。在未来的构建过程中，不要让自己陷入混乱；放松一下，听听专业人士给你的一些整理东西的建议。

### **了解你的库存**

看一看你的工作台。你看到了什么？过去项目的碎片会阻碍今天的进展吗？对诺亚来说不是。通过将他的项目本地化到一个单一的混乱中，他为自己节省了无数的时间，并且通过将大量使用的重复部件装入那个盒子，他将为自己节省更多的时间。诺亚还发誓要让他的项目箱保持 30%的开放，准备好容纳他以后会捡起来的想法。

### 任务日志

你最后一次告诉你的朋友或老板:“哦，对了，那个 LED 矩阵面板是什么时候？两天后就会亮起来”…两周后(或者两个月后)却发现自己深陷在一些疯狂的不稳定问题或线路故障中？估计完成下一个项目需要多长时间是一件棘手的事情，但是诺亚给了我们一个窍门:*检查你的时间*。无论是 Google-Doc 还是文本文件，记录修复的错误、订购的组件以及下一个周末留给机器处理的部件是帮助我们更好地了解完成一个项目需要多长时间的关键。更重要的是，我们可以清楚地知道我们从哪里停下来，然后重新投入到项目中。得心应手；尤其是当我们在等待邮件中更多的部分时，我们正在处理另一个项目。

### 在我把自己弄成什么样之前，我是什么样的人？

再看看那个工作台。很有可能你会得到不止一个，而是三个或四个半成品项目。我们怎样才能足够努力地穿过最后几英里的沙砾来消灭他们呢？诺亚告诉我们从头开始，问自己:“那个项目中的多巴胺在哪里？”当我们眨第一个 LED 时，我们的眼睛会发亮吗，即使我们还有一百个要布线，或者我们会结束今天的工作，把它扔进废品箱吗？诺亚承认，如果没有这种动力，如果我们在项目过程中没有任何收获，就很难说服自己完成项目。如果我们是项目背后的头脑和双手，我们将需要享受其中的每一点，所以在做出长期承诺之前，先寻找那些时刻。

### 通过项目在道路上嵌入真理

诺亚在过去的十年里一直在做东西。从美国国家航空航天局的第一个 fab 实验室，到诺亚用于动态纹理变化表面实验的开放平台 [WEFT](https://hackaday.io/project/8446-weft-electrovibration-demo-board) ，诺亚一直在砂砾中进进出出，学习什么最能让他成功。他的一些做法有可衡量的好处；其他的不要。但是，即使他没有量化打扫他的家庭商店或整理他的无线模块所节省的时间，诺亚是生活在制造东西的空间中的*，他希望你欢迎你的到来。*
