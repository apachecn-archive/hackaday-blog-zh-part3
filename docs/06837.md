# 超级会议采访:本·克拉斯诺

> 原文：<https://hackaday.com/2017/08/16/superconference-interview-ben-krasnow/>

本·克拉斯诺是一个完美的原型。他建造了一台制造完美巧克力曲奇的机器，他有一台红宝石激光器，他自己在内部生产液氮，因为简单地装满一个杜瓦瓶太容易了。如果你需要一个原型，本就是你要找的人。

Ben [在去年的 Hackaday Superconference](https://www.youtube.com/watch?v=S-pxPcF2zUQ) 上做了一个关于快速原型制作和验证技术假设的演讲。哲学可以简单地总结为“先建设，后提问”。当他想看看反向散射 x 射线机是否真的比 TSA 检查站的金属探测器更有效时，这种哲学很好地服务了他。回答这个问题通常的方法是找一个 x 光专家，等待几周，支付几万美元，最终得到答案。本只是用从易贝采购的零件建造了自己的反向散射 x 光机。

 [https://www.youtube.com/embed/lPpSit4Gofg?version=3&rel=1&showsearch=0&showinfo=1&iv_load_policy=1&fs=1&hl=en-US&autohide=2&wmode=transparent](https://www.youtube.com/embed/lPpSit4Gofg?version=3&rel=1&showsearch=0&showinfo=1&iv_load_policy=1&fs=1&hl=en-US&autohide=2&wmode=transparent)



演讲结束后，我们向 Ben 询问了这种先建设后提问的哲学的局限性。有了本所拥有的身体和精神工具集，实际上很容易构建出能够回答一个问题的东西。问题来了，当本需要证明一些*不会*工作。

回答这个问题完全是心态问题。在本看来，如果一个原型起作用，一个假设就被验证了。即使这完全是个意外，他也完全接受这个结果。他的一些其他同事有一个相反的心态——如果一个快速和肮脏的原型不起作用，一个研究假设得到验证。

这种快速验证概念的心态是我们在普通观众中经常看到的，我们知道你们中有些人的头脑和车库至少和本一样令人印象深刻。我们已经延长了 2017 年黑客日超级大会的[提案征集时间](https://docs.google.com/forms/d/e/1FAIpQLSfcq-ZWoK0Jj-xY_Lcx15oT_zmWwCF-eCX9BbpnvaZAp4bpEw/viewform)。如果你有一个关于快速成型或者只是用机器人制作完美的巧克力曲奇的故事，我们想听听。[11 月 11 日和 12 日在加州帕萨迪纳市举行的超级大会的门票仍然有售。](https://www.eventbrite.com/e/hackaday-superconference-2017-tickets-35735832838)
