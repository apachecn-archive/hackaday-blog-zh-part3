# FlowCode 图形编程

> 原文：<https://hackaday.com/2017/04/20/flowcode-graphical-programming/>

如果你曾经好奇是否有一种不用实际写软件就能编程微控制器的方法，你可能会对 FlowCode 感兴趣。这不是一个免费的产品，但有一个免费的演示。[Web learning]使用系统做了一个编程 Nucleo 板的[演示。你可以在下面查看一下。](https://www.youtube.com/watch?v=a6FQDfC6Ys4)

该产品看起来很光滑，它支持令人眼花缭乱的处理器数量，从 AVR(是的，它会做 Arduino)，PIC 和 ARM targets。然而，如果您实际上想要针对所有这些处理器，价格可能会增加，因为您最终要为 CPU 和组件付费。例如，非商业入门包的价格约为 75 美元，支持一些流行的处理器和组件，如 led、PWM、旋转编码器等。

撇开价格不谈，我们非常担心用 GUI 构建应用程序。这听起来很诱人，对于简单的项目来说，这很简单也很直观。然而，当你有流程图时，复杂的问题会很快变得混乱。这也是复杂逻辑设计远离原理图(另一种图形表示形式)而转向 Verilog 和 VHDL 的原因。

不过，看看这是如何做到的可能会给你一些想法，它可能只是正确的应用程序的门票。当然，这不是城里唯一的图形游戏。比如有 [ArduBlock](https://hackaday.com/2011/11/30/graphical-programming-with-the-arduino-ide/) 。另一个我们已经看过的是[迷你烧烤](https://hackaday.com/2014/05/08/minibloq-ide-graphical-programming-for-robots-of-all-sizes/)。你甚至可以在 Arduino 或 Raspberry Pi 上使用 [Scratch](https://hackaday.com/2015/10/08/latest-raspberry-pi-images-have-improved-scratch/) 。
