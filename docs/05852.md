# JeVois 机器视觉相机指甲演示模式

> 原文：<https://hackaday.com/2017/05/17/jevois-machine-vision-camera-nails-demo-mode/>

[JeVois](http://jevois.org/) 是一款小型开源智能机器视觉相机，于 2017 年初在 Kickstarter 上获得融资。我支持它，因为嵌入机器视觉元素的相机正在稳步发展，功能越来越强，JeVois 拥有令人印象深刻的一系列功能。它运行嵌入式 Linux，可以使用 OpenCV 算法以高帧率处理视频。它可以独立运行，也可以作为 USB 摄像头将原始或预处理的视频传输到主机，以供进一步操作。在任一情况下，它都可以通过串行端口与其他设备通信(并受其控制)。

但当我收到我的单元时，这些都不是相机真正打动我的地方。真正突出的是演示模式。JeVois *背后的团队为复杂的设备找到了有效的演示模式。那不是偶然发生的，结果值得分享。*

## 一个好的演示的重要性

当涉及到复杂系统时，一个好的演示模式本质上是对单元能力的电梯式推销。对于用户，它回答“这能做什么，它为我打开了什么可能性？”

JeVois 相机的演示模式通过做几件事成功做到了这一点:

1.  **使演示内容完备，易于开始。**只需用户提供最少的零件或设置即可开始。将系统映像放入附带的 SD 卡后，我只需要将它插入我的笔记本电脑并启动相机浏览器。
2.  使它具有互动性。立即响应用户输入，尽可能展示工作中的流程。
3.  保持简单。演示并不是解释一切的唯一机会！让用户自由地专注于吸收正在显示的内容；避免让用户陷入解决界面或问题的困境。

如果存在的话，硬件上的演示模式通常是事后才想到的，但是它值得关注和改进，因为它是产品的一个元素，几乎可以肯定每个用户都会使用它。

## JeVois 的设置和演示

我必须将系统映像复制到 micro SD 卡上，以确保我拥有最新版本的 JeVois 软件，但之后我需要做的只是将它插入 USB 端口(我的计算机将它识别为 USB 摄像头设备)并打开网络摄像头浏览器。一旦完成，演示就自动开始了。硬件将循环“巡回”视频与现场摄像机图像以及大量并行运行的视觉过程混合在一起。对于主机操作系统来说，它只是一个来自 USB 摄像头的视频流。

不是每个元素都完美无瑕；尤其是手写数字识别，虽然演示也承认这一点，但它总是时好时坏。下面的演示视频是在一个有阳光的白色房间里录制的，但事情仍然看起来暗淡。最后，相机和镜头显然不打算改变或调整。我更喜欢带 M12 ( [S-mount](https://en.wikipedia.org/wiki/S-mount_(CCTV_lens)) )镜头的模块，但 JeVois 上的相机至少有一个非常小的封装。不管怎样，这个演示成功地简单明了地展示了可能的东西。
