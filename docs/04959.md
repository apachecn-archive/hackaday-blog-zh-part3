# 运动检测摄像机使用云识别人类

> 原文:[https://hack aday . com/2017/02/06/motion-detecting-camera-recognizes-human-using-the-cloud/](https://hackaday.com/2017/02/06/motion-detecting-camera-recognizes-humans-using-the-cloud/)

[马克·韦斯特]和他的妻子有一个问题，他们有不速之客在他们的花园里。马克的解决方案是想出一个动作激活的安全摄像头系统，当有人在花园里走动时，它会向他发送电子邮件。没错，只有人类。从技术角度来看，更有趣的是，他在云中做了大量的处理工作。他给云发送了一张照片，照片上有东西在动，只有当照片上有人类时，他才发送电子邮件。

[Mark]的第一次迭代在他的网站上描述得非常好，它包括将所有现成的组件组装在一起，包括 Raspberry Pi Zero、Pi NoIR 相机和 ZeroView 相机支架，使他可以轻松地将其安装在他的窗户内侧，俯瞰花园。他使用[动作](https://github.com/Motion-Project/motion)来检查相机的图像，寻找任何有运动的帧。每次检测到运动时，他的代码都会给他发一封附有照片的电子邮件。问题是，有几天他会收到多达 50 个误报的电子邮件提醒:移动的影子、邻居的猫，甚至是落在窗户上的雨水。

这引导他进行下一次迭代，在照片中寻找人类。为此，他选择将照片传递给亚马逊网络服务(AWS)在线工具来检查人类。

但那留下了一个决定。他可以将包含运动的图像发送到 Rekognition，获得结果，然后给自己发送一封关于包含人类的图像的电子邮件，或者他可以将图像发送到云，让云与 Rekognition 对话，然后让云给他发送电子邮件。他选择让云来做这件事，这样 Pi 零和云组件就可以更好地解耦，使得对任何一个组件的更改都更容易。

这里没有足够的空间来讲述他是如何在云上做到这一点的所有细节，所以我们将把它留给[【马克的】详细报道](https://utbrudd.bouvet.no/2017/01/10/smarten-up-your-pi-zero-web-camera-with-image-analysis-and-amazon-web-services-part-2/)。但可以说，这是一本非常有趣的读物。最有趣的是这是可能的。这意味着使用处理器受限的微控制器的黑客不仅可以使用在线服务完成复杂的人工智能任务，还可以卸载一些中间任务。
