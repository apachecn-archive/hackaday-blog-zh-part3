# 星期五黑客聊天:为开源开发做贡献

> 原文:[https://hack aday . com/2017/12/06/Friday-hack-chat-contributed-to-open-source-development/](https://hackaday.com/2017/12/06/friday-hack-chat-contributing-to-open-source-development/)

开源是世界运行的方式。在你现在所坐的思维盒子深处的某个地方，有你可以自己查看、修改、编译和运行的代码。在你的路由器和给你发送这个网页的可怕的 WordPress 服务器之间的路径上的每一点，都有传输字节的开源位。如果没有开源软件，我们所知道的世界就不会存在。

也就是说，一个人如何对开源做出贡献？维护者确实喜欢建立自己的小王国，那么怎么会有人闯入开发开源硬件和软件呢？

[![](../Images/cc863c8079c6bce88fc40dfddf3c7930.png)](https://hackaday.com/wp-content/uploads/2017/12/contributingtoopensourcehackchat-01.png) 我们这次 Hack Chat 的嘉宾是 Robert Wolff，他是一名技术作家，也是一名开源福音传道者，曾在 STE*M 教育项目中工作过。现在，罗伯特是利纳罗 96 个董事会的社区经理。96 板是一种硬件规范，旨在以合理的成本提供最新的 ARM 处理器。这一开放式规范为 SoC 无关平台定义了标准的电路板布局，系统软件开发人员可以使用任何应用、设备和内核。

在这次黑客聊天中，我们将关注的问题是如何为开源项目做出贡献，如何使用 96 板做到这一点，记录开放系统所涉及的技术挑战，设计处理器无关系统的困难，以及关于 96 板社区、生态系统和资源的一般问题。

一如既往，我们将接受来自 hackaday.io 社区的提问，因此，如果您有任何问题，[请将问题提交到 Hack Chat 活动页面](https://hackaday.io/event/28431-contributing-to-open-source-development)。
