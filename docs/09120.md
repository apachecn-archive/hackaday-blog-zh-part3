# 星期五黑客聊天:与社区讨论 MQTT

> 原文:[https://hack aday . com/2018/04/11/Friday-hack-chat-talking-mqtt-with-the-community/](https://hackaday.com/2018/04/11/friday-hack-chat-talking-mqtt-with-the-community/)

物联网就在眼前，无论如何，所有这些智能灰尘和果汁袋压榨机将不得不彼此交谈。让物联网比特彼此交谈的更好方法之一是 MQTT，*消息队列遥测传输*，这是一种为小代码足迹和有限网络带宽设计的协议。它得到了很多物联网炒作，但对于你自己的小项目来说，它是 HTTP 的一个很好的替代选择，所以这就是我们在本周的黑客聊天中谈论的[。](https://hackaday.io/event/112092-mqtt-community-chat)

[![](../Images/d521ff71a33177b03d8eb0a75e8f87c3.png) ](https://hackaday.com/wp-content/uploads/2018/04/mqtt-hack-chat.png) MQTT 是一种机器对机器的连接协议，在远程位置非常有用，在这些位置需要很少的代码，带宽非常宝贵，或者当坐在同一间屋子里时用手机打开和关闭灯。它是移动应用程序的理想之选，在自其诞生以来的二十年左右的时间里，MQTT 已经进入了您家中所有的“智能”设备。

MQTT 基于一个非常简单的带有“主题”的发布和订阅模型，允许您配置消息应该发送到哪里。这是一个非常简单的协议，但是有了 MQTT，你可以[建立一个完整的家庭自动化系统，打开车库门，打开一盏灯，或者 pings 几个天气传感器](https://hackaday.com/2016/05/09/minimal-mqtt-building-a-broker/)。

在本周的 Hack Chat 中，我们将与整个 Hackaday.io 社区讨论 MQTT。有几十个构建了基于 MQTT 的项目的人经常参加 Hack Chat，还有几百个想要学习。想进入物联网的底层吗？这是你想看的黑客聊天。这是一个围绕互联设备的社区盛会。
